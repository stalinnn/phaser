# 全息混合双轨制 RAG (Holo-Hybrid Dual-Track RAG) 实验总结与下一步计划

## 一、 当前实验结果汇总 (Phase 1 总结)

我们在“如何将高维欧氏特征投射到低维双曲空间”这一方向上进行了深度的探索，经历了以下几个关键的实验阶段：

### 1. 尝试 1：基础 MLP 投影 + Margin Loss
*   **方法**：使用 BGE-M3 (1024D) 提取特征，经过简单的线性 MLP 映射到 64D 双曲空间，使用 Margin Ranking Loss 训练。
*   **结果**：**严重失败**。发生了“维度崩塌/特征坍塌”，所有向量挤向双曲空间的边缘，模型失去了区分正负样本的能力（MRR 极低）。

### 2. 尝试 2：引入 InfoNCE Loss 与 黎曼优化
*   **方法**：更换损失函数为 InfoNCE（对比学习，含 in-batch hard negatives），引入 Riemannian Adam 优化器。
*   **结果**：**短暂回光返照**。初期评估得出了较高的 MRR (0.94+)，但随后被证明这只是基于极其狭窄的数据集（严重的数据泄露）得出的幻象。在更严谨的同等维度测试下，**64D 双曲 (0.77) 甚至打不过 64D 欧氏 (0.82)**。

### 3. 尝试 3：强制层级约束 (Hierarchy Loss) 与 蒸馏 (Distillation)
*   **方法**：为了解决“维度崩塌”，我们强行加入了基于范数半径的 `Hierarchy Loss`，要求 Query（问题）更靠近球心，Context（上下文）更靠近边缘。同时加入 `KL Distillation` 以保留欧氏语义。
*   **结果**：**赢了理论，输了实战**。
    *   几何形状完美符合预期：Query 平均半径 0.52，Context 平均半径 0.63。
    *   检索性能却剧烈暴跌：1024D 欧氏(0.84) -> 加了新Loss的双曲(0.52)。
    *   **核心原因**：MLP 这座“弱桥梁”无法同时承载“保留语义”和“重塑拓扑”两个矛盾的任务。模型为了满足人工设定的几何层级，彻底碾碎了精细的语义特征。这印证了 The Bitter Lesson。

### 4. 尝试 4 (突破)：裸拓扑极限压缩测试 (Naked Topology Extreme Test)
*   **方法**：彻底抛弃文本内容和 BGE-M3 特征。只根据数据集里的 QA 连接关系（图结构），在极端的 **4维空间** 下，从随机初始化开始训练欧氏节点和双曲节点。
*   **结果**：**双曲空间的完胜**。
    *   4D 欧氏 MRR：0.9741（由于维度空间挤满，陷入瓶颈，错误率 2.6%）
    *   4D 双曲 MRR：**0.9980**（凭借指数级膨胀的边缘体积，完美容纳图结构，错误率 0.2%）
    *   **终极结论**：双曲流形天生就是 RAG 树状/图状结构的完美容器。我们之前的失败，纯粹是因为“把字面语义和逻辑拓扑强行绑定在一个空间内转换”的策略错误。

## 二、 实验代码梳理与清理

基于上述结论，我们清理了 Phase 1 阶段所有基于“MLP 语义扭曲投影”的失败代码，保留并重构了以下核心资产库：

### 留存的核心资产：
1.  **`6_naked_topology_test_hard.py`**：这是奠定我们新方向的基石脚本。它证明了纯拓扑图在双曲空间的优越性。
2.  **`curvature_detector.py` / `geom_rag_router/detector.py`**：流形曲率探测器。用于在运行时判断问题是属于“平直事实”还是“树状多跳”，这是未来“智能路由”的大脑。
3.  **`1_extract_embeddings.py`**：BGE-M3 基础特征提取（用于后续的主炮）。
4.  **`evaluate_beir_datasets.py` 等 BEIR 相关脚本**：用于在 MS MARCO 和通用数据集上评估欧氏主炮的零样本能力。

### 已废弃/删除的死胡同代码：
*   `2_train_holo_embedder.py`（旧版带各种纠结 Loss 的投射训练脚本）
*   `3_evaluate_hybrid_rag.py`（基于旧版模型的评估）
*   `5_compare_losses.py`（旧路线下的内耗对比脚本）
*   `6_naked_topology_test.py`（发生“天花板效应”的32D容余版测试）

## 三、 下一步计划：迈向 Phase 2 (演生全息 RAG - Emergent Holographic RAG)

既然“语义属于平直欧氏，逻辑属于弯曲双曲”，我们将彻底抛弃“用MLP强行扭曲欧氏语义”的做法。真正的全息对偶（AdS/CFT）不是没有映射的平行双轨，而是**“内部弯曲时空的几何结构，完全是由边界平直时空的量子纠缠（相似度）自发涌现出来的”**。

我们将引入 **Ryu-Takayanagi (RT) 映射准则**（边界纠缠熵=内部极小曲面面积），构建一个真正的演生时空架构。

### 1. 引擎构建与全息映射
*   **边界层：平直时空主炮 (Semantic Euclidean Engine)**
    *   直接使用未微调的 BGE-M3 (1024D)。
    *   **作用**：它代表边界上的共形场论（CFT），负责计算节点间的“语义纠缠度（余弦相似度）”，并为检索提供“冷启动锚点”。
*   **内部层：弯曲时空副炮 (Topological Hyperbolic Engine)**
    *   **非局域全息映射 (Holographic Dictionary)**：在极低维度（如 16D Poincaré Ball）下训练知识图谱节点。我们不再用 MLP 做点对点映射，而是通过映射**度量 (Metric)** 让双曲空间自发涌现：强制要求双曲空间中两点的**测地线距离**，完美拟合它们在边界平直欧氏空间网络中的**连通性与语义分布**（甚至可以通过图卷积HGCN聚合邻居特征实现区域积分映射）。
    *   **作用**：基于主炮提供的锚点，在内部双曲空间沿着极小曲面（测地线）进行球面扩散和协变输运，抗噪地捞出处于深层逻辑树上的多跳答案。

### 2. 路由与融合 (Routing & Late Fusion)
*   **开发曲率路由器 (Curvature Router)**：整合 `curvature_detector.py`。
    *   低曲率问题：走主炮即可。
    *   高曲率问题（多跳）：主炮定锚点 -> 唤醒副炮 -> 双曲扩散。
*   **设计非线性融合算法 (Poincaré-RRF)**：
    *   彻底抛弃简单的线性倒数排名相加。
    *   探索基于双曲测地线距离去指数衰减欧氏得分的新融合函数：`Final_Score = Euc_Score * exp(-alpha * Hyp_Dist)`。

- [ ] **Model Dev**: 基于 `6_naked_topology_test_hard.py` 的成功经验，开发一个可用于百万级节点规模的 16D 双曲图节点嵌入训练器（考虑引入负采样的优化以加速大规模训练）。
