# EvTTC 全双工流体控制引擎 (Full-Duplex Fluid Control for EvTTC)

**项目代号**: Project Fluid-Drive
**目标场景**: 基于事件相机 (Event Camera) 的毫秒级自动驾驶碰撞预警 (Time-To-Collision)
**理论底座**: 物理人工智能大统一理论 (GUT of Physics-AI) - 连续统智能场与非平衡态热力学

## 1. 为什么选择 EvTTC 场景？
传统的自动驾驶视觉系统（基于传统摄像头与 CNN/ViT）面临致命的“帧延迟 (Frame Latency)”与“批处理 (Batch Processing)”困境：系统必须等待数十毫秒采集完一帧图像，再花费算力在全局计算特征。在高速行驶时，这数十毫秒的盲区可能导致不可逆的灾难。

事件相机 (DVS) 以兆赫兹的频率异步输出像素亮度的**对数梯度变化 (Events: x, y, t, p)**，这与我们的**全双工流体网络**天生契合：
- **无帧 (Frameless)**：取消数据打包，事件如水滴般连续流入系统。
- **动态演化 (Dynamic Evolution)**：隐状态（势能场）无需全局复位，而是在 Langevin 动力学驱动下发生局部连续热化。
- **抢答预警 (Pre-emptive Alert)**：一旦前车急刹（导致局部事件流密度激增），系统势能场瞬间打破平衡，触发雪崩效应，在碰撞前数毫秒**持续辐射**出刹车指令，而非等待下一帧计算。

## 2. 核心架构设计 (The Architecture)

系统包含三个核心组件，所有组件均不保留随时间增长的 KV Cache，实现 **O(1) 恒定极低显存占用**。

### 2.1 连续流体泵 (Continuous Event Pump)
将异步事件流映射为对高维隐状态（势能场）的局部扰动梯度。
- 不再是对整张图片的卷积，而是针对发生事件的极少数像素点进行稀疏的状态注入。

### 2.2 黑洞热化引擎 (Langevin Thermalization Engine)
模型的主体。用常微分方程/偏微分方程的离散形式替代传统的 Transformer Layer。
- **耗散项**：随时间自然衰减（遗忘陈旧的安全路况）。
- **引力项**：提取碰撞特征（如扩张的光流场）。
- **热噪声**：防止陷入局部极小值。

### 2.3 熵坍缩辐射器 (Entropy-Collapse Radiator)
实时监控系统隐状态的局部信息熵。
- 当熵较高时（路况安全、无明显特征），系统保持“沉默”或输出极低置信度的信号。
- 当事件流构成明确的“碰撞特征（迫近的障碍物）”，系统熵迅速坍缩，像黑洞辐射一样连续高频输出 `[TTC: 0.5s! 紧急制动]` 信号。

## 3. 工程目录结构
- `data/`: 存放 EvTTC 事件流数据切片 (H5/Numpy 格式)。
- `models/`: 
  - `fluid_core.py`: 连续统智能场的核心演化方程定义。
  - `event_encoder.py`: 异步事件的稀疏注入器。
- `demo_evttc_fluid.py`: 可视化演示脚本，用于直观对比传统“按帧推理”与本项目的“全双工流体抢答”。
- `train_fluid_ttc.py`: 模型训练脚本（采用热力学 Loss）。

## 4. 下一步计划 (Roadmap)
1. 编写核心流体网络算子 `FluidCore`，实现无 Cache 的隐状态演化。
2. 开发极其震撼的可视化 Demo：用动画展示事件星点如何激起模型内部的热力图涟漪，并在毫秒级引发报警辐射。
3. 接入真实的 EvTTC 数据进行微调，验证 O(1) 显存和极端低延迟（Latency < 5ms）的降维打击能力。
